# DNS의 작동 원리
DNS는 도메인을 IP 주소로 변환해주는 시스템입니다.
브라우저는 먼저 캐시를 확인하고, 없으면 로컬 DNS 서버에 질의합니다.
로컬 서버도 모르면 root -> TLD -> 권한 DNS 서버 순으로 재귀 질의를 통해 IP를 찾습니다.
응답받는 IP는 브라우저가 사용하고, 이후 요청 속도를 높이기 위해 모두 캐싱됩니다.

## 전체 흐름
1. 브라우저에서 도메인 입력
2. 브라우저 캐시 확인
3. 운영체제 캐시 확인
4. 로컬 DNS 서버로 질의 
5. 재귀적 질의 시작
   - [Root DNS 서버] -> [Top Level DNS 서버] -> [Authoritative DNS 서버]
6. IP 응답 및 캐싱
