인터넷은 전세계 컴퓨터가 서로 통신할 수 있게 해주는 네트워크 입니다.
작동 흐름은 크게 다음과 같습니다.
1. 도메인 요청 : 사용자가 브라우저에 도메인을 입력합니다.
2. DNS 조회 : 도메인을 IP 주소로 변환하기 위해 DNS 서버를 조회합니다.
3. 패킷 전송 : 변환된 IP 주소로 TCP/IP 프로토콜 기반의 패킷이 전송됩니다.
4. 라우팅 : 패킷은 여러 라우터를 거쳐 목적지 서버로 전달됩니다.
5. 서버 응답 : 서버는 클라이언트에게 HTML, CSS, JS 등의 데이터를 응답합니다.
6. 렌더링 : 브라우저는 응답을 바탕으로 화면을 랜더링합니다.

요약
인터넷은 클라이언트와 서버가 TCP/IP 기반으로 데이터를 주고받는 구조입니다.
사용자가 도메인을 입력하면, DNS를 통해 해당 도메인의 IP 주소를 찾고, 패킷은 라우터들을 거쳐 서버에 도달합니다.
서버는 HTTP 또는 HTTPS를 통해 응답을 보내고, 클라이언트는 이를 렌더링해 사용자에게 웹페이지를 보여줍니다.
전체 흐름은 도메인 -> DNS -> TCP/IP -> 서버 요청 및 응답 -> 렌더링 순 입니다.

추가 질문
- TCP와 UDP의 차이
- DNS는 재귀적 질의와 반복적 질의 방식 존재
- HTTPS는 TLS를 통해 보안성 확보
