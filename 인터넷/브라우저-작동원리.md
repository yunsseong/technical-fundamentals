# 브라우저의 작동 원리
브라우저는 사용자가 URL을 입력하면 서버에 요청을 보내고 HTML을 받아옵니다.
HTML을 파싱해 DOM을 만들고, CSS와 결합해 렌더 트리를 구성합니다.
그 후 레이아웃 계산과 페인팅 과정을 거쳐 사용자에게 웹 페이지를 보여줍니다.
이후 자바스크립트를 통해 동적 동작이나 사용자 입력을 처리하게 됩니다.

## 전체 흐름
1. URL 입력 & 요청 전송
- 사용자가 주소창에 URL을 입력
- 브라우저는 DNS를 통해 IP 주소를 알아내고, HTTP(S) 요청을 서버로 전송

2. 서버 응답 수신
- 서버가 HTML 문서를 응답
- 브라우저는 HTML을 분석하며, 추가로 필요한 리소스(CSS, JS, 이미지 등)도 병렬로 요청

3. HTML 파싱 & DOM 트리 생성
- HTML을 파싱해 DOM(Document Object Model) 트리 생성
- 요소 간의 계층 구조를 메모리에 구조화

4. CSS 파싱 & 렌더 트리 생성
- CSS 파일도 파싱하여 CSSOM(CSS Object Model) 생성
- DOM과 CSSOM을 결합해 렌더 트리(Render Tree) 생성

5. 레아아웃 & 페인팅 & 컴포지팅
- 레이아웃 : 각 요소의 크기와 위치 계산
- 페인팅 : 요소들의 시각적 표현 준비 (색상, 이미지 등)
- 컴포지팅 : 레이어를 하나로 합쳐 최종 화면을 그림

6. 이후 자바스크립트 실행, 사용자 입력 처리
- JS 실행 시 DOM/CSSOM 변경 -> 재랜더링 (Repaint / Reflow)
- 이벤트 루프를 통해 사용자 입력 처리
