# HTTP란 무엇인가?
HTTP는 웹에서 클라이언트와 서버 간 데이터를 주고 받기 위한 프로토콜입니다.
비연결성, 무상태성을 특징으로 하며, 텍스트 기반 요청/응답 형식을 사용합니다.
주로 HTML, 이미지 같은 웹 리소스를 주고받을 때 사용되며, HTTPS는 여기에 보안을 추가한 버전입니다.

# 특징
HTTP의 비연결성과 무상태성은 서버 자원을 아끼고, 대규모 사용자 처리를 가능하게 해줍니다.
요처이 끝나면 연결을 끊고, 상태를 저장하지 않기 때문에 서버는 단순하고 확장성이 높으며, 로드밸런싱과 장애 대응에 유리합니다.

## 비연결성 (Stateless)
- 한 번의 요청/응답 이후 연결을 유지하지 않음
- 매 요청은 독립적으로 처리됨
### 중요한 이유
- 자원 효율성 : 서버가 모든 클라이언트와 계속 연결을 유지하려면 많은 메모리와 CPU가 필요함
- 대규모 확장에 유리 : 동시에 수천~수만 명이 접속해도 서버는 요청 단위로 처리하면 되므로 확장성 높음

## 무상태성
- 서버는 클라이언트의 이전 요청 상태를 기억하지 않음
- 상태 유지를 위해 쿠키나 세션을 사용함
### 중요한 이유
- 매 요청이 독립적이기 때문에 복잡한 세션 관리 없이 요청만 처리하면 됨
- 모든 서버가 같은 요청에 대해 같은 응답을 할 수있으므로, 로드밸런싱이 쉬움
- 상태 저장이 없기 때문에, 서버 간 전환이나 복구가 간단함
